## 目的

タブレット（Windows + Edge + ペン）で画譜PDFを閲覧・管理し、ページ単位でスワイプ操作によりページめくりでき、手書きで注釈を書き込めるブラウザアプリを提供する。ネットワークが無い場所でも使用できる（PWA + 端末内保存）。

## 対象環境 / 前提

- **端末**: Windows タブレット
- **ブラウザ**: Microsoft Edge
- **入力**: ペン入力（Pointer Events）
- **オフライン**: 初回セットアップ後はネットワーク無しで利用可能

## スコープ

### スコープ内

- **PDF取り込み**: ローカルPDFを読み込み、曲として保存
- **管理画面（トップ）**: 保存済み曲の一覧・選択・削除
  - 一覧表示は **PDFファイル名のみ**
- **閲覧画面**:
  - PDFを **1ページずつ表示**
  - **横スワイプ（左右）**で前後ページ移動
  - **手書き注釈**（黒/赤/青、太さ3段階）
  - **消しゴム**
  - **Undo/Redo**
- **永続保存**:
  - 一度読み込んだPDFは端末内に記憶
  - 注釈も端末内に保存
- **オフライン対応**: PWA（Service Worker キャッシュ + IndexedDB）

### スコープ外（当面）

- クラウド同期 / 複数端末同期
- ユーザーアカウント
- PDFの編集（ページ削除/結合等）
- 高度な注釈（テキスト付箋、図形、画像貼り付け）
- 曲名/タグ等のメタ情報編集（今回はファイル名のみ）

## 画面設計

### 1) トップ（管理画面）

- **曲一覧**（カード or リスト）
  - 表示: `filename`
  - 操作:
    - **開く**（閲覧画面へ）
    - **削除**
- **PDF追加**ボタン
  - `file input` でローカルPDFを選択→読み込み→IndexedDBに保存

### 2) 閲覧画面

- **中央**: PDFページ表示領域
- **注釈レイヤ**: PDFの上にCanvas（透明）を重ね、ペン描画
- **最小UIバー**:
  - 戻る（トップへ）
  - ファイル名
  - ページ番号（例: `3 / 12`）
- **ツールパレット**:
  - ペン色: 黒/赤/青
  - 太さ: 3段階
  - 消しゴム
  - Undo / Redo
- **ジェスチャ**（演奏モード = 閲覧画面）:
  - 横スワイプで前後ページ
  - **左右端タップゾーン**（タッチのみ）:
    - 楽譜画像表示領域の**左端から30%幅**をタップ → 前ページ
    - **右端から30%幅**をタップ → 次ページ
    - 中央40%はタップ無効（誤操作防止）
    - 縦方向は表示領域の全高
  - ペン描画中はスワイプ／タップ判定を抑制（誤ページ送り防止）

## データモデル（IndexedDB）

### ストア

- `scores`
  - `id: string`（uuid）
  - `filename: string`
  - `createdAt: number`
  - `updatedAt: number`
  - `pageCount: number`
- `pdfs`
  - `scoreId: string`
  - `pdfBytes: ArrayBuffer`（元PDF）
- `annotations`
  - `scoreId: string`
  - `page: number`（0-based）
  - `updatedAt: number`
  - `strokes: Stroke[]`

### 注釈形式

`Stroke` は以下の最小構成とする。

- `tool: "pen" | "eraser"`
- `color?: "black" | "red" | "blue"`（penのみ）
- `width: 1 | 2 | 3`（3段階を内部値にマップ）
- `points: { x: number; y: number; t: number }[]`

**座標はページ座標に正規化**する。基準は「PDF.js の viewport（scale=1）で得た width/height」とし、保存時はその座標系で保持する。表示時は現在のviewportへアフィン変換して描画する。これにより端末解像度やズームで注釈がズレない。

### Undo/Redo

- 単位: **1操作 = 1ストローク追加 / 1ストローク削除（消しゴムによるヒット）**
- 各ページに `undoStack` / `redoStack` を持つ（閲覧中のみメモリ上）
- 操作確定後に `annotations` を永続化

### 消しゴムの仕様（最小）

- 消しゴムは「ポインタ軌跡に一定半径を持たせ、その領域と交差するストロークを削除」する
- 削除対象の判定は、ストロークの点列と消しゴム軌跡の距離が閾値以下（例: 画面pxで一定）になったらヒットとみなす
- ヒットしたストロークはストローク単位で削除（部分消しは当面スコープ外）

## PDF描画

- PDF.js でページをレンダリングし、1ページずつ表示する
- ページの描画キャンバスと注釈キャンバスを重ねる
- ページ変更時に該当ページの注釈を読み込み、注釈キャンバスへ再描画

## 入力とジェスチャ（スワイプ／タップ）

- Pointer Events を利用する（ペン/指を統一的に扱う）
- **ペン（pointerType="pen"）** は基本「描画」のみとし、スワイプ／タップ判定を行わない
- **タッチ（pointerType="touch"）** は横スワイプ／左右端タップでページ送りを行う

### 座標の基準

- タップゾーンの基準は **PDFページ表示領域（PDF描画キャンバス＝注釈キャンバスが重なる矩形）の幅**
- ゾーン境界は左端から 0〜30% を `prev`、70〜100% を `next`、30〜70% の中央40% はタップ無効
- 縦方向は表示領域の全高を対象

### 判定の優先順（pointerdown → pointerup）

1. **スワイプ判定**（既存）: 水平移動量が閾値以上かつ垂直移動量が小さい場合に `prev/next` を確定
2. **タップ判定**: スワイプ不成立 かつ `pointerup` までの水平／垂直移動量がともにタップ閾値（例: 10px）以内の場合、
   - `pointerdown` 開始位置が左ゾーン（0〜30%）→ `prev`
   - 右ゾーン（70〜100%）→ `next`
   - 中央ゾーン → 何もしない
3. 上記いずれにも該当しない場合は無視（描画は開始しない）

### 誤動作防止

- ペン接触中（描画中）はスワイプ／タップ判定を行わない
- UIバー／ツールパレットは表示領域の外側に置き、タップゾーンの対象外とする

## オフライン（PWA）

- アプリ本体（HTML/CSS/JS/アイコン/フォント）をService Workerでキャッシュ
- ライブラリ（PDF.jsなど）も同様にキャッシュ
- 曲データと注釈はIndexedDBに保存されるため、オフラインで閲覧/書き込み可能
- 初回インストール/キャッシュ構築のため **初回のみネットが必要**（以後オフライン可）

## エラーハンドリング / UX

- PDF読み込み失敗時: 「読み込みに失敗しました」+ 再試行
- 大きいPDFの初回読み込み: 進捗/スピナー表示
- ストレージ不足: 容量不足メッセージ + 不要な曲の削除導線
- 誤スワイプ防止: 描画中（ペン接触中）はページ送りジェスチャを無効化

## 成功条件（受け入れ基準）

- ネット無しの環境でアプリが起動し、保存済みの曲を開いて閲覧できる
- PDFが1ページずつ表示され、横スワイプでページ送りできる
- 楽譜画像の **左端から30%幅をタップ → 前ページ**、**右端から30%幅をタップ → 次ページ** に移動できる（中央40%のタップは無反応）
- ペンで注釈が描け、色3種・太さ3段階が反映される
- 消しゴムとUndo/Redoが動作する
- アプリを閉じて再起動しても、読み込んだPDFと注釈が保持される
